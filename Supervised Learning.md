# 📝 머신러닝 기초 공부 노트

## 1. 과적합 (Overfitting)과 모델의 상태
* **정의:** 모델이 훈련 데이터(Training Data)를 너무 달달 외워서, 실제 시험(새로운 데이터)에서는 엉뚱한 답을 내는 상태.
* **특징:** 훈련 데이터 정확도는 100%에 가깝지만, 검증/테스트 데이터 성능은 뚝 떨어짐 (**높은 분산, 낮은 편향**).
* **해결 방법:**
    * **데이터 증강 (Augmentation):** 데이터를 변형해 양 늘리기
    * **드롭아웃 (Dropout):** 딥러닝에서 뉴런을 무작위로 끄며 학습
    * **조기 종료 (Early Stopping):** 검증 손실이 올라가기 직전에 학습 멈추기
    * **가중치 규제:** 특정 가중치가 너무 커지지 않도록 페널티 부여 ($L_1$, $L_2$ 규제)

## 2. 오차 행렬 (Confusion Matrix)과 4대 평가 지표
> **용어 암기 팁:** 뒤에서부터 읽기! (예: False Positive $\rightarrow$ 양성(P)이라고 예측했는데 틀렸다(F) $\rightarrow$ 실제는 음성)

### 📌 4가지 기본 상태
* **TP (True Positive):** 양성 예측 $\rightarrow$ 맞음 (실제 양성)
* **TN (True Negative):** 음성 예측 $\rightarrow$ 맞음 (실제 음성)
* **FP (False Positive):** 양성 예측 $\rightarrow$ 틀림 (실제 음성) -> *1종 오류*
* **FN (False Negative):** 음성 예측 $\rightarrow$ 틀림 (실제 양성) -> *2종 오류*

### 📌 핵심 성능 평가 지표
1.  **정확도 (Accuracy)**
    * $$\text{Accuracy} = \frac{\text{TP} + \text{TN}}{\text{전체}}$$
    * 가장 직관적이지만, 데이터 비율이 불균형할 때 사기를 치기 쉬움.
2.  **정밀도 (Precision)**
    * $$\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}}$$
    * **가로축 기준:** 모델이 "양성"이라고 예측한 것 중 진짜 양성의 비율 (예: 스팸 메일 분류).
3.  **재현율 (Recall / Sensitivity)**
    * $$\text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}$$
    * **세로축 기준:** 실제 양성 데이터 중 모델이 찾아낸 비율 (예: 암 진단, 코로나 검사).
4.  **F1 스코어 (F1-Score)**
    * $$F_1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$
    * 정밀도와 재현율의 **조화 평균**. 두 지표가 균형을 이뤄야만 점수가 높음 (**데이터 불균형 해결의 치트키**).

## 3. 로지스틱 회귀 (Logistic Regression)
* **정의:** 이름은 회귀이지만, 실제로는 **이진 분류(0 또는 1)**를 수행하는 알고리즘.
* **원리:** 선형 회귀 결과($wx+b$)를 **시그모이드 함수(Sigmoid Function)**에 통과시켜 0과 1 사이의 확률값으로 변환.
    * $$\sigma(z) = \frac{1}{1 + e^{-z}}$$
* **결정 경계 (Decision Boundary):** 출력된 확률이 보통 `0.5` 이상이면 1(양성), 미만이면 0(음성)으로 분류.

## 4. 앙상블 학습 (Ensemble Learning)
* **정의:** 여러 개의 단일 모델(약한 예측기)을 결합하여 하나의 강력한 예측 모델을 만드는 집단지성 기법.



### 📌 3대 알고리즘 유형
* **배깅 (Bagging):** 데이터를 중복 허용하여 샘플링 후, 여러 모델을 **병렬**로 학습시켜 투표/평균으로 통합 (과적합 방지, 대표작: **랜덤 포레스트**).
* **부스팅 (Boosting):** 앞선 모델이 틀린 오답에 가중치를 주어 **순차적(직렬)**으로 학습 (성능이 극대화되지만 과적합 위험 있음, 대표작: **XGBoost, LightGBM**).
* **보팅/스태킹 (Voting/Stacking):** 서로 다른 종류의 알고리즘 결과를 다수결로 정하거나(보팅), 예측 결과를 다시 입력 데이터로 삼아 최종 모델로 재학습(스태킹).

## 5. 핵심 분류 모델 비교 및 추천 (Logistic vs Tree vs Forest)

머신러닝 프로젝트를 진행할 때 데이터의 특성과 목적에 따라 어떤 모델을 선택해야 하는지 정리한 표입니다.

| 모델명 | 주요 장점 | 주요 단점 | 추천 활용 상황 |
| :--- | :--- | :--- | :--- |
| **로지스틱 회귀**<br>(Logistic Regression) | • 속도가 매우 빠름<br>• 모델이 단순하고 직관적<br>• 가중치 규제가 쉬움 | • 비선형 데이터(복잡한 경계)에 취약함 | • 프로젝트 초기의 **Baseline(기본 기준점)** 모델<br>• 데이터가 선형적으로 깔끔하게 나뉠 때 |
| **의사결정나무**<br>(Decision Tree) | • 시각화가 가능하며 매우 직관적<br>• 데이터 전처리(스케일링) 영향이 적음 | • 가지치기를 안 하면 **과적합(Overfitting)**되기 너무 쉬움 | • 분석 결과를 비전문가나 고객에게 **설명(시각화)해야 할 때** |
| **랜덤 포레스트**<br>(Random Forest) | • 여러 나무를 합친 앙상블 기법으로 **성능이 매우 좋음**<br>• 과적합에 강함 | • 모델이 무거워 학습/예측 속도가 상대적으로 느림 | • **실무/현업** 프로젝트 및 데이터 분석 대회(Kaggle 등)<br>• 안정적인 고성능이 필요할 때 |

---

### 💡 모델 선택 가이드라인 (Workflow)

1. 데이터를 처음 만나면 가장 먼저 가볍고 빠른 **Logistic Regression**을 돌려서 기준 점수(Baseline)를 잡습니다.
2. 만약 데이터의 인과관계나 규칙을 눈으로 직접 확인하고 누군가에게 쉽게 설명해야 한다면 **Decision Tree**를 사용해 시각화합니다.
3. 설명력보다는 무조건 정답을 잘 맞추는 최종 예측 성능을 끌어올려 실무에 적용해야 한다면, Tree 모델들을 앙상블한 **Random Forest**나 부스팅 모델(LightGBM, XGBoost)을 최종 선택합니다.
